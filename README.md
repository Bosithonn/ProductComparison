# Product Comparison Widget

React + TypeScript приложение для сравнения товаров. Тестовое задание.

---

## Что умеет

- Просматривать список из 8 ноутбуков
- Выбирать до 3 штук для сравнения
- Таблица сравнения: характеристики по строкам, товары по столбцам
- Строки, где значения различаются, визуально выделены
- Убирать любой товар из сравнения
- Выбранные товары сохраняются при перезагрузке страницы

---

## Стек

- React + TypeScript (Vite)
- Чистый CSS — без UI-библиотек
- localStorage для сохранения состояния

## Почему я принял такие решения

**Без Context API и сторонних библиотек состояния**

Выбранное состояние — это просто массив максимум из 3 ID. Прокинуть его пропсами из App вполне достаточно, никакой глубокой вложенности нет. Тащить Zustand или Context ради трёх строк стейта — это избыточно. Всю логику вынес в хук `useComparison`, чтобы App.tsx оставался чистым, а работа с localStorage была в одном месте.

**localStorage через ленивую инициализацию**

Вместо того чтобы читать localStorage в useEffect (что даёт моргание при маунте), я инициализирую стейт лениво:

```ts
const [ids, setIds] = useState<string[]>(() => {
  const saved = localStorage.getItem("cmp_ids")
  return saved ? JSON.parse(saved) : []
})
```

Так состояние правильное уже на первом рендере, никакого флика пустого экрана.

**Чистый CSS**

Не захотел тянуть Tailwind или библиотеку компонентов для такого небольшого проекта. Когда масштаб маленький, обычный CSS читается проще и не раздувает бандл. Использовал два файла — `index.css` для сброса стилей и body, `App.css` для всего остального.

**Логика подсветки различий**

Для каждой строки характеристик собираю все значения из выбранных товаров и проверяю, одинаковые ли они. Если нет — строка получает класс с подсветкой. Простая проверка через `Set`: если `new Set(values).size > 1`, значения разные.

**Максимум 3 товара**

Если уже выбраны 3, клик на четвёртый просто ничего не делает. Никакого тоста, модалки или алерта. Три заполненных колонки в таблице и так дают достаточно контекста. При желании можно добавить тултип, но для этого задания посчитал лишним.
